# Анализ клиентских данных интернет-магазина

## 📋 Описание проекта

В этом проекте я провёл анализ данных интернет-магазина на основе датасета **Online Retail** (транзакции за 2010–2011 гг.). Цель — исследовать поведение клиентов, оценить удержание и пожизненную ценность. 
Файл с анализом данных, графиками и выводами - online_retail.ipynb. Для просмотра можно открыть в git, нажав на него.

**Данные:** [Online Retail Dataset на Kaggle](https://www.kaggle.com/datasets/vijayuv/onlineretail)

## 📊 Основные этапы

1. **Очистка данных:** удаление дубликатов, пропусков, обработка возвратов, создание датафрейма продаж.
2. **EDA:** анализ выручки, сезонности, географии, топ-товаров и размера заказов.
3. **Когортный анализ:** расчёт Retention (удержания) по месяцам.
4. **LTV-анализ:** расчёт средней и накопленной выручки на клиента по когортам.

## 🔍 Ключевые результаты

### 1. Общие метрики
- **Валовая выручка:** 8.9 млн
- **Количество заказов:** 18 536
- **Количество клиентов:** 4 339
- **Средний чек:** 479.46
- **Медианный чек:** 302.56

### 2. Когортный анализ Retention
- **Лучшая когорта:** декабрь 2010 (Retention на 1-й месяц – 36.6%, пик – 50.3% на 11-й месяц)
- **Средний Retention на 1-й месяц:** ~22–23%

### 3. LTV
- **Средний LTV:** 674
- **Максимальный LTV:** 6636 (выброс, клиент с аномально крупной покупкой)
- **Накопленный LTV лучшей когорты (2010-12):** 12 494 за 12 месяцев


## 💡 Рекомендации для бизнеса

- **Усилить маркетинг в ноябре–декабре** — именно в эти месяцы привлекаются самые лояльные клиенты.
- **Для анализа LTV использовать медиану** (а не среднее), так как выбросы искажают картину.
